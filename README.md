Запуск проекта осуществляется с помощью программы Docker. Ссылка для загрузки Docker: https://www.docker.com/products/docker-desktop/

1) Форкнуть репозиторий проекта: Vladislav-st-1984/Kittygram_v2.0

1.1) Откройте страницу нужного репозитория на GitHub — найдите проект, который хотите форкнуть.

1.2) Найдите кнопку Fork. Она расположена в правом верхнем углу страницы, рядом с количеством звёзд у проекта (иконка с двумя ответвляющимися ветками).

1.3) Нажмите на кнопку Fork. Откроется выпадающее меню с опциями.

1.4) Выберите настройки форка:

Owner — выберите, куда поместить форк: ваш личный профиль или организацию, в которой вы состоите. По умолчанию выбран ваш аккаунт.

Repository name — задайте имя для форка. По умолчанию оно совпадает с оригиналом, но вы можете изменить.

Description — опционально обновите описание репозитория.

Copy the main branch only — поставьте галочку, если хотите скопировать только основную ветку (например, main или master), а не все ветки проекта.

1.5) Нажмите Create fork. Начнётся процесс создания форка — обычно он занимает несколько секунд.

1.6) Дождитесь завершения. После окончания вы автоматически перейдёте на страницу вашего нового форка — это полная копия оригинального репозитория в вашем аккаунте.

2) Клонировать форкнутый репозиторий командой:

-- git clone https://github.com/ваш-username/Kittygram_v2.0.git --

3) Сборка и запуск проекта (все действия выполняются в терминале Windows "PowerShell" в директории проекта)

Для сборки проекта используется команда:

-- docker-compose build --no-cache backend frontend --

Для запуска проекта используется команда:

-- docker-compose up --

Проект находится по адресу:  http://localhost:9000/signin

Для перезапуска и пересборки проекта используются команды:

Для остановки:

-- docker-compose down --

Для сборки:

-- docker-compose build --no-cache backend frontend --

Для запуска:

-- docker-compose up --

4) Для добавления миграций:

-- docker-compose exec backend python manage.py migrate --

5) Для создания администратора:

-- docker-compose exec backend python manage.py createsuperuser --
